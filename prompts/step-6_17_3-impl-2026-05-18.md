## Locked decisions

1. **POST is multi-audience.** Diverges from tenants POST (`audience="PLATFORM"`). Stores POST allows TENANT-tier OWNER to create stores in own tenant. `tenant_id` in body verified against caller's RLS-bound session. No `audience=` kwarg on the gate. Owner: design 2026-05-18.

2. **`store_code` and `tax_treatment` required at schema layer.** DDL stays nullable; NOT NULL migration deferred to future step after cloud testing. Owner: operator decision 2026-05-18.

3. **`org_node_id` is immutable on PATCH.** Schema rejects `org_node_id` in body via `extra="forbid"`. DDL allows mutability; app-layer restriction is conservative pending product workflow decision. Future loosening is additive (backward-compatible). Owner: operator decision 2026-05-18.

4. **PATCH empty body → 422 EMPTY_PATCH; non-empty same-as-current → 200 no-op.** Mirrors tenants and tenant-users exactly. `updated_at` bumps on any non-empty PATCH even when values are no-ops. Owner: codebase precedent.

5. **`store_code` uniqueness is per-tenant, app-layer pre-checked.** SELECT-then-INSERT pattern, mirroring tenants `DuplicateTenantName`. Pre-check excludes self on PATCH rename. App-layer enforcement; DDL has no UNIQUE constraint. Race window accepted at v0 scale. Owner: design 2026-05-18.

6. **`org_node_id` validity enforced by DDL composite FK + partial unique index; app-layer pre-check exists for typed-error 409.** Without pre-check, conditions surface as `IntegrityError → 500`. With pre-check, typed `OrgNodeNotForStoreError` (409). Owner: design 2026-05-18.

7. **Audit columns populated from `AuthContext`.** POST: all four audit-actor columns (`created_by_*` + `updated_by_*`) set from `auth.user_id` and `auth.user_type`. `created_at` + `updated_at` from DDL default. PATCH: `updated_by_*` set; `updated_at` re-assigned in UPDATE statement. `closed_*` untouched (6.17.4 owns). Owner: codebase precedent.

8. **Status server-forced on POST.** `status` rejected in POST body via `extra="forbid"`. New stores start at DDL default `OPENING`. Server does not override. Owner: 6.17-root design.

9. **Pattern (b) audit-actor population: BOTH `*_by_user_id` AND `*_by_user_type` columns populated on every write.** `core.stores` is a Pattern (b) table per D-13. Repo signatures take `actor_user_id: UUID` + `actor_user_type: ActorUserType` as separate kwargs. INSERTs populate created_by_* AND updated_by_* pairs. UPDATEs populate updated_by_* pair. `ck_stores_*_actor_pair` CHECKs enforce both-NOT-NULL invariants. The handler uses `_actor_type_from_auth(auth)` to bridge `AuthContext.user_type` (Literal) to `ActorUserType` (enum). Owner: codebase precedent (Step 6.10.1, 6.13).

```
class StorePatchRequest(BaseModel):
    name: str | None = Field(default=None, min_length=1, max_length=200)
    store_code: str | None = Field(default=None, min_length=1, max_length=50)
    country: str | None = Field(default=None, min_length=2, max_length=100)
    timezone: str | None = Field(default=None, min_length=1, max_length=50)
    currency: str | None = Field(default=None, pattern=r"^[A-Z]{3}$")
    tax_treatment: TaxTreatment | None = None
    address: str | None = None
    latitude: Decimal | None = Field(default=None, ge=-90, le=90)
    longitude: Decimal | None = Field(default=None, ge=-180, le=180)

    model_config = ConfigDict(extra="forbid")
```

`extra="forbid"` enforces immutability. POST body permits the 7 required fields plus 4 optional (org_node_id, address, latitude, longitude); excludes id, status, all audit columns, all closed_* columns. PATCH body permits the 9 mutable fields; excludes id, tenant_id, org_node_id, status, all audit columns, all closed_* columns.

NUMERIC-as-string serialisation on latitude/longitude mirrors the `TenantRead.monthly_revenue_usd` precedent (field_serializer with `when_used="json"`). Applies only to response shape, not request parsing - Pydantic handles request-side string-to-Decimal coercion natively.

### errors.py (MODIFY: +2 classes)

```
class DuplicateStoreCodeError(ClientError):
    http_status = 409
    code = "DUPLICATE_STORE_CODE"

    def __init__(self, tenant_id: UUID, store_code: str):
        super().__init__(
            message=f"Store code '{store_code}' already exists in tenant {tenant_id}",
            tenant_id=tenant_id,
            store_code=store_code,
        )


class OrgNodeNotForStoreError(ClientError):
    http_status = 409
    code = "ORG_NODE_NOT_FOR_STORE"

    def __init__(self, org_node_id: UUID, reason: str):
        super().__init__(
            message=f"Org node {org_node_id} cannot be linked: {reason}",
            org_node_id=org_node_id,
            reason=reason,
        )
```

`reason` is a server-side string for diagnostics; not surfaced to API consumers as machine-parseable. Three causes the pre-check might raise:
- `"not found"` (no row at that id)
- `"different tenant"` (org_node exists but belongs to a different tenant)
- `"already linked"` (org_node already has another store linked via the partial unique index)

### repositories/stores.py (MODIFY: +2 methods)

`core.stores` is a Pattern (b) audit-actor table per D-13 (per `architecture_RBAC.md`: "Tables using Pattern (b) audit-actor columns (per D-13: `tenant_users`, `org_nodes`, `stores`, `roles`, `role_permissions`, both `*_role_assignments` tables)"). Repo signatures take `actor_user_id: UUID` + `actor_user_type: ActorUserType` as separate kwargs, mirroring `TenantUsersRepo.create` and `TenantUsersRepo.update` exactly. INSERT and UPDATE statements MUST populate BOTH columns of each `*_by_user_*` pair to satisfy `ck_stores_created_by_actor_pair` and `ck_stores_updated_by_actor_pair`.

```
class StoresRepo:
    # ... existing list, get_by_id ...

    async def create(
        self,
        session: AsyncSession,
        *,
        tenant_id: UUID,
        name: str,
        country: str,
        timezone: str,
        currency: str,
        store_code: str,
        tax_treatment: TaxTreatment,
        org_node_id: UUID | None,
        address: str | None,
        latitude: "Decimal | None",
        longitude: "Decimal | None",
        actor_user_id: UUID,
        actor_user_type: ActorUserType,
    ) -> StoreDetailRow: ...

    async def update(
        self,
        session: AsyncSession,
        store_id: UUID,
        *,
        fields: dict[str, Any],
        actor_user_id: UUID,
        actor_user_type: ActorUserType,
    ) -> StoreDetailRow | None: ...
```

**`create` body** (~30 lines):

1. If `org_node_id` is not None: run pre-check (composite SELECT to verify (a) org_node exists, (b) belongs to `tenant_id`, (c) not already linked to another store). Raise `OrgNodeNotForStoreError` with appropriate `reason` on failure.
2. Run pre-check for `store_code` uniqueness within `tenant_id` (SELECT-then-INSERT pattern). Raise `DuplicateStoreCodeError` on collision.
3. INSERT into `core.stores` with all kwargs + audit-actor pair (BOTH `created_by_user_id` + `created_by_user_type` AND `updated_by_user_id` + `updated_by_user_type` populated from the actor kwargs). Cast `:actor_user_type` to `actor_user_type_enum` in the SQL (per the Pattern (b) reference example).
4. Return joined Store + tenant_name row (mirror `StoreDetailRow` shape from 6.17.2's `get_by_id`).

Schema-qualify all raw SQL (`core.stores`, `core.org_nodes`).

**`update` body** (~25 lines):

1. SELECT existing row (RLS-bound). Return None if not found.
2. If `fields` includes `store_code` and the new value differs from current: run `DuplicateStoreCodeError` pre-check excluding self (`WHERE tenant_id = ? AND store_code = ? AND id != ?`).
3. UPDATE `core.stores` SET <fields> + `updated_by_user_id = :actor_user_id` + `updated_by_user_type = CAST(:actor_user_type AS actor_user_type_enum)` + `updated_at = NOW()` WHERE id = store_id. Use `RETURNING *` or follow up with the JOIN to materialize the return row.
4. Return the updated row in `StoreDetailRow` shape.

Note: `update` does NOT validate `org_node_id` since the schema rejects it; the field cannot reach this layer.

Helper convention: the handler uses the existing `_actor_type_from_auth(auth: AuthContext) -> ActorUserType` helper (defined in `repositories/tenant_users.py` per Step 6.10.1; check if exported from there or if a local re-declaration is preferred; the convention is "small helper at the top of each Pattern (b) repo file"). Surface during impl if the helper's location requires a new import path.

**OpenAPI descriptions** (verbatim spec for codegen):

- POST endpoint docstring: "Create a new store. Multi-audience. PLATFORM tier (Super Admin, Platform Admin) creates for any tenant via GLOBAL cascade. TENANT tier (Owner) creates for own tenant via direct TENANT grant; tenant_id in body verified against RLS-bound session. Status server-forced to OPENING via DDL default. store_code uniqueness checked per-tenant before INSERT (app-layer; future DDL migration tightens). org_node_id optional; when provided, must reference an org_node in the same tenant not already linked to another store."

Empty body returns 422 EMPTY_PATCH. Rename of store_code to a taken value returns 409 DUPLICATE_STORE_CODE; rename to same as current is a 200 no-op. updated_at bumps on any non-empty PATCH."
