## DDL facts (verified against current_schema.sql at 2026-05-16)

### core.org_nodes

- PK: `id UUID DEFAULT core.uuidv7() NOT NULL`.
- Composite UNIQUE `uq_org_nodes_tenant_id` on `(tenant_id, id)`: D-34 composite-FK target.
- UNIQUE index `uq_org_nodes_tenant_code_lower` on `(tenant_id, lower(code))`: tenant-wide case-insensitive code uniqueness. The arbiter for LD10's 409 mapping.
- FKs: `tenant_id` to `tenants(id)`; `parent_id` to `org_nodes(id)` self-referencing.
- `path public.ltree NOT NULL`: full materialized path. Repo maintains app-side.
- `node_type core.org_node_type_enum NOT NULL`: 7 values listed in LD4.
- `status core.org_node_status_enum NOT NULL DEFAULT 'ACTIVE'`: 3 values (`ACTIVE`, `INACTIVE`, `ARCHIVED`). 6.13 only writes `ACTIVE`.
- Five CHECK constraints:
  - `ck_org_nodes_root_parent_consistency`: `(node_type='TENANT' AND parent_id IS NULL) OR (node_type!='TENANT' AND parent_id IS NOT NULL)`. DDL backstop for LD5.
  - `ck_org_nodes_code_format`: `^[A-Za-z0-9][A-Za-z0-9-]{0,62}[A-Za-z0-9]$` OR length=1. 1-64 chars, alphanumerics with hyphens, must start/end alphanumeric, no underscores.
  - `ck_org_nodes_name_length`: 1-200 chars.
  - `ck_org_nodes_created_by_actor_pair` + `ck_org_nodes_updated_by_actor_pair`: Pattern (b) audit-actors (user_id + user_type pair).
  - `ck_org_nodes_archived_consistency`: pertains to ARCHIVED state, not touched by 6.13.
- Indexes: `ix_org_nodes_parent`, `ix_org_nodes_path_gist` (GiST on `path`), `ix_org_nodes_tenant`, `ix_org_nodes_tenant_type`.
- Trigger: `tg_org_nodes_set_updated_at` BEFORE UPDATE FOR EACH ROW. Sets `updated_at` only. Repo must explicitly set `updated_by_user_id` and `updated_by_user_type` on every UPDATE.
- RLS: ENABLE + FORCE. Policy `org_nodes_tenant_isolation` standard D-29 OR-branch.

## Transition contract

| Operation | Body shape | Outcome |
|---|---|---|
| POST (Add Node) | `{parent_id, node_type, code, name}` | INSERT new row, `status=ACTIVE`, `path=parent.path || lower(code).replace("-","_")`, audit columns from caller. Cascade-order checked. Returns 201 + OrgNodeRead. |
| PATCH (rename only) | `{name?, code?}` | UPDATE row's name/code; path untouched; audit columns updated. Returns 200 + OrgNodeRead. |
| PATCH (reparent only) | `{parent_id}` | SELECT FOR UPDATE on target and new_parent. Cascade-order + cycle checks. UPDATE target's `parent_id` + `path`. Re-path all descendants in one statement. Returns 200 + OrgNodeRead. |
| PATCH (combined) | `{name?, code?, parent_id}` | All of the above atomically. Returns 200 + OrgNodeRead. |
| PATCH on tenant root with `parent_id` | `{parent_id: <something>}` | 422 TENANT_ROOT_NOT_REPARENTABLE. |
| PATCH all fields None | `{}` | 422 (Pydantic validator: at least one mutable field required). |
