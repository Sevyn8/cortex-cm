## Locked decisions

**LD1: Pattern shape is the multi-label ltree path regex.**

```
pattern = r"^[A-Za-z0-9_]+(\.[A-Za-z0-9_]+)*$"
```

Matches Postgres ltree label grammar: alphanumerics + underscore per label, dot-separated. Single-label-only would reject valid live paths like `cornerstop.cs_main` and `buc_ees.bu_hq.tx.tx_101.deli`.

Source: investigation Bucket 3c + live data inspection.

**LD2: Inline `pattern=` on the Query declaration, not a shared type.**

Mirrors the 4 existing pattern validators in the codebase (schemas/org_node.py:271, 334; schemas/store.py:153, 187). Shared `LtreePath` Pydantic type is over-investment for a single call site.

Source: investigation Bucket 3a + FN-AB-61 option (c) trigger (no second user-supplied-ltree endpoint exists today).

**LD3: max_length = 1024.**

Postgres ltree per-label cap is 256 chars; full path historically 65535 bytes. Local seed's deepest path is 31 chars. Conservative cap of 1024 covers any realistic org-tree depth without leaking the implementation cap or imposing artificial limits.

Source: investigation Bucket 3c.

**LD4: Bug only fires under TENANT JWT.**

Per investigation Bucket 1b: `_has_permission_platform` (auth/permissions.py:241-282) does NOT reference `:target_anchor`. Only `_has_permission_tenant` (lines 284-344) reaches the ltree CAST. Cloud incident at v0.1.17 was a TENANT JWT. Tests must cover the TENANT path specifically.

Source: investigation Bucket 1b.

**LD5: Docstring correction in schemas/org_node.py:274-275.**

Existing docstring says "No underscores (ltree label restriction)" which is backwards: ltree labels REQUIRE underscores; org_node code convention forbids underscores and uses hyphens. Fix to: "No underscores (org_node code convention; underscores are converted to hyphens via _path_label for ltree storage)."

Source: investigation Bucket 3b finding + Bucket 5 observation.

**LD6: Test shape mirrors `test_v7_invalid_code_format_pydantic_422`.**

Single test function with multiple assertion blocks for each failure shape. Negative inputs: hyphen-bearing, whitespace, leading-dot, trailing-dot, consecutive-dots, empty-string. All assert 422 from Pydantic envelope.

Source: investigation Bucket 4b precedent at tests/integration/test_org_tree_writes_router.py:485-521.
