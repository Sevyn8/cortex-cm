## Scope in (this commit)

Add a single optional query parameter `actor_user_id: UUID | None = None` to the existing GET `/api/v1/audit/activities` list endpoint. The parameter filters audit rows by the actor's user_id, AND-composed with existing filters, applied to both UNION branches of the audit-logs repo query. No `actor_user_type` companion parameter needed (UUIDs are globally unique across `platform_users.id` and `tenant_users.id`; `actor_user_id` alone is fully selective).

Closes FN-AB-69 (frontend drawer Activity tabs need an actor filter to consume `/audit/activities` instead of the dead `/audit-logs` endpoint).

Bundle one design doc edit (per A6 / LD9):
- `docs/architecture_audit_logs.md` Read contract > Filter parameters table: add one row for `actor_user_id`.
