## Scope in (this commit)

Audit row schema extension + emission retrofit + GET endpoint response schema extension to support the audit list-view redesign (Phase 1 + Phase 2 of 6.16.7).

**DDL changes (both audit tables):**
- New column `actor_organization_name TEXT NOT NULL` (Path A: add NULLABLE, backfill, then SET NOT NULL).
- New column `actor_roles TEXT NOT NULL` (same Path A treatment).
- New column `resource_subtype TEXT NULL` (no backfill needed; ORG_NODE rows only, pre-6.16.7 historical rows render with fallback).

**Emission retrofit (all 16 sites):**
- Every audit emission path resolves and populates the 3 new columns at write time.
- Actor role resolution: JOIN `tenant_user_role_assignments` (or `platform_user_role_assignments`) → `core.roles` for active roles; aggregate `roles.name` (display values, not `roles.code`); comma-separated string (e.g., "Owner, Promotions Assistant"). UI-renderable shape; no further frontend transformation.
- Actor organisation resolution: tenant name for tenant actors via JOIN on `tenants`; literal `"Platform-Ithina"` for platform actors.
- Resource subtype resolution: only for ORG_NODE emission paths; read `org_nodes.node_type` at write time.

**Vocabulary changes:**
- `_label_for_action`: UPDATE label changes from "Updated" to **"Edited"**; SET_STATUS label changes from "Status change" to **"Set status"**.
- New backend dispatch: 9 CONFLICT error classes → per-class qualifier phrases; `result_label` composes `"Blocked – <qualifier>"`.

**GET endpoint response schema (additive only):**
- `AuditActivityListItem` grows from 8 fields → 14 fields. 6 new additive fields: `actor_organization_name`, `actor_roles`, `what`, `resource_type`, `resource_subtype`, `result_type`. All existing 8 fields kept unchanged in shape.
- Backend composes `what` field at read time: `"Type: Name"` shape from `resource_type` + `resource_subtype` (for ORG_NODE rows) + `resource_label`.

**Documentation bundled per A6:**
- `docs/architecture_audit_logs.md`: Schema section (3 new column rows on both tables), Read contract section (6 new response fields documented), new "Display vocabulary" subsection (action labels + CONFLICT qualifiers + ORG_NODE subtype labels).
- `BUILD_PLAN.md`: 6.16.7 root entry (single-commit step).

**FN-AB impact:**

- FN-AB-68 unchanged (OPEN_SOFT stays dormant; Phase 1 locked).
- New FN-AB: INTEGRITY_VIOLATION reserved vocabulary; no callers; revisit when use case surfaces.
