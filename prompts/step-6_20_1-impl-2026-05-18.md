## Context (the bug being fixed)

`POST /api/v1/tenants` (Step 6.11.2) inserts rows into `core.tenants` + `core.tenant_module_access` but does NOT insert a tenant-root row into `core.org_nodes`. The seed loader DOES insert this row (one per seeded tenant), and every endpoint gated with `anchor_dep=get_tenant_anchor` (e.g. GET `/api/v1/tenants/{id}`, PATCH, suspend, activate, org-tree, org-nodes/children, module-access enable/disable) raises `TenantNotFoundError` 404 when the anchor lookup misses. Result: tenants created via POST are unreachable through every detail / write endpoint.

The contract that every tenant has exactly one `(node_type='TENANT', parent_id IS NULL)` org_node is already documented (e.g. `OrgNodeCreateRequest` description: "tenant roots are provisioned with the tenant") and assumed by every consumer. This step makes `TenantsRepo.create` honour that contract.

Orphan cleanup on local + cloud was completed pre-fix (10 cloud orphans + 1 local orphan deleted in a single transaction each, 2026-05-18).

## Scope out (deferred with triggers)

- **Backfill org_nodes for production tenants.** Not needed - cleanup deleted all orphans pre-fix. Trigger: if a future production env shows orphan tenants, run the one-off cleanup SQL captured in this step's Retro.

- **Slug-conflict resolution beyond truncation.** v0: truncate at 64 chars, accept that two very long names sharing a 64-char prefix would collide on `uq_org_nodes_tenant_code_lower`. Structurally unreachable at tenant-root insert (no other nodes exist yet in the tenant), so the collision can only happen if two tenant-creates with identical name prefixes race within the same transaction-isolation window, which is itself prevented by the existing `_raise_if_name_taken` pre-check on `tenants.name`. FN-AB captures the risk for later revisit. Trigger: if collision ever surfaces in practice.

- **Editorial short-codes for POST-created tenants.** Seed tenants have curated codes (e.g. `Buc-ee's` → `BUC-EES`, `Żabka Group` → `ZABKA`) that drop words / strip diacritics in non-mechanical ways. POST-created tenants get a mechanical slug (Buc-ee's → `BUC-EES`, Żabka Group → `ZABKA-GROUP`). Editorial override deferred to a future PATCH-on-tenant-root surface. Trigger: product request.

- **Tightening regex / validation rules on `code`.** v0 mechanical slug already produces conformant codes; DDL CHECK `ck_org_nodes_code_format` is the backstop. No additional Pydantic validators on the request body. Trigger: empirical name-shape data showing non-Latin tenant names becoming problematic.

## Locked decisions

**LD1: `display_code` stays optional on `TenantCreateRequest`.** Making it required would be a contract break on the existing endpoint; operator confirmed (Phase 1 Q1 2026-05-18). When omitted, derive from `name`. When present, derive from `display_code` directly. Owner: operator decision Phase 1 Q3.

**LD2: org_node insert sits between `tenants` INSERT and `tenant_module_access` per-module loop in `TenantsRepo.create`.** Order: (1) name uniqueness pre-check (existing) → (2) `tenants` INSERT (existing) → (3) NEW: tenant-root `org_node` INSERT → (4) per-module `tenant_module_access` loop (existing) → (5) fetch aggregated row (existing). All steps inside the single request transaction inherited from FastAPI session dep; atomic by construction. Precedent for org_node insert shape: Step 6.13 `OrgNodesRepo.add_node` (raw SQL, schema-qualified, audit-actor Pattern (b)). Owner: operator confirmation Phase 1 Q7.

**LD3: `(code, path)` derivation rule.** Mechanical slug; ASCII-only output.

Input: `display_code` if present, else `name`.

Steps:
1. lowercase
2. strip diacritics (NFKD decompose, drop combining marks) - e.g. `Żabka` → `Zabka` → `zabka`
3. replace runs of non-alphanumeric chars with single `-`
4. trim leading/trailing `-`
5. truncate to 64 chars; if truncation lands on `-`, trim again
6. if result is empty → raise `InvalidTenantNameForSlugError` (422)
7. `code = result.upper()`
8. `path = result.replace('-', '_')`

Examples (locked):
- `Buc-ee's` (no display_code) → slug `buc-ee-s` → code `BUC-EE-S`, path `buc_ee_s`
- `Buc-ee's` (display_code=`buc-ees`) → slug `buc-ees` → code `BUC-EES`, path `buc_ees`
- `Żabka Group` (no display_code) → slug `zabka-group` → code `ZABKA-GROUP`, path `zabka_group`
- `Fresh Test Co` (no display_code) → slug `fresh-test-co` → code `FRESH-TEST-CO`, path `fresh_test_co`
- `!!!` (no display_code) → empty after slug → 422 `INVALID_TENANT_NAME_FOR_SLUG`

Note: this is intentionally simpler than the seed shape. POST-created Buc-ee's would get code `BUC-EE-S` not `BUC-EES` because the seed-curated dropping of `'s` is an editorial decision, not a transform rule. This is documented in Scope out.

Owner: operator decisions Phase 1 Q3, Q4, Q5.

**LD4: empty-slug case raises 422.** New error class `InvalidTenantNameForSlugError` (code `INVALID_TENANT_NAME_FOR_SLUG`, 422). Raised by the slug helper before the `tenants` INSERT runs. Public message names the field that produced the empty slug (`name` or `display_code`). Owner: operator decision Phase 1 Q5.

**LD5: no slug-collision retry logic at tenant-root insert.** The `uq_org_nodes_tenant_code_lower` UNIQUE is `(tenant_id, lower(code))` - tenant-scoped. At tenant-root insert, the tenant has no other nodes, so collision within the new tenant is structurally impossible. Owner: operator confirmation Phase 1 Q6.

**LD6: helper location.** Pure-function helper `slug_for_tenant_root(name: str, display_code: str | None) -> tuple[str, str]` lives in `src/admin_backend/repositories/tenants.py` as a module-level function (not a method on the class). Rationale: no DB dependency, pure-functional, unit-testable in isolation. If a second resource needs the same slug rule later, promote to `utils/` then. Owner: drafter decision; surface if codebase precedent suggests otherwise.

**LD7: audit-actor Pattern (b) for `org_nodes` insert.** Per Step 6.13 DDL facts: `org_nodes.created_by_user_id` + `created_by_user_type` form a Pattern (b) pair (DDL CHECK `ck_org_nodes_created_by_actor_pair`). At `POST /api/v1/tenants` the actor is the JWT's PLATFORM user, so `created_by_user_id = actor_user_id`, `created_by_user_type = 'PLATFORM'`. Same for `updated_by_*`. Owner: operator confirmation Phase 1 Q8.

**LD8: tenant-root org_node row shape.**

- `id` - DB default `core.uuidv7()`
- `tenant_id` - the new tenant's id from step 2
- `parent_id` - NULL (DDL CHECK `ck_org_nodes_root_parent_consistency` requires this for `node_type='TENANT'`)
- `node_type` - `'TENANT'`
- `code` - from slug helper
- `name` - `name` (the tenant's name; matches seed shape - `Buc-ee's` tenant has org_node name `Buc-ee's`)
- `path` - from slug helper (ltree-castable; underscores not hyphens)
- `status` - `'ACTIVE'`
- `created_by_user_id` / `created_by_user_type` - actor + `'PLATFORM'`
- `updated_by_user_id` / `updated_by_user_type` - same
- Other audit columns - DB defaults (`created_at`, `updated_at` from `now()`)
- `archived_*` columns - NULL

**1. Add module-level helper:**

```python
def slug_for_tenant_root(
    name: str,
    display_code: str | None,
) -> tuple[str, str]:
    """Derive (code, path) for tenant-root org_node.

    See LD3 for the rule. Returns uppercased code and underscore-separated
    ltree path. Raises InvalidTenantNameForSlugError if slug is empty.
    """
```

Behavior spec:
- input = `display_code` if not None else `name`

- strip leading/trailing `-`
- truncate to 64; re-strip trailing `-` if truncation landed on one

- return `(s.upper(), s.replace('-', '_'))`

### Deliberately not added

- **No test asserting `org_nodes.code` matches an exact uppercase form of `display_code`.** The slug rule is unit-tested directly in `tests/unit/test_tenant_root_slug.py`; duplicating those assertions inside an integration test would be a brittle change-detector. The integration tests assert the slug is "consistent with the helper output" by re-calling the helper, not by hardcoding strings.
- **No test asserting the ordering of writes inside the transaction.** Transaction ordering is implementation detail; what matters is atomicity (rollback-on-error) and final state. `test_create_empty_slug_rejects_no_tenant_inserted` covers the rollback case.
- **No test for collision on `uq_org_nodes_tenant_code_lower` at tenant-root insert.** Structurally unreachable per LD5; testing it would require manually corrupting the DB state ahead of the call. FN-AB-NN captures the residual risk for future revisit.
- **No DDL regression test for `ck_org_nodes_root_parent_consistency`.** Already covered by Step 6.13's test suite for the org-tree router; this step inherits the coverage.

## Appendix A: architecture.md content to apply at 6.20.1

### Tenant create transaction shape (Step 6.20.1)

`POST /api/v1/tenants` (PLATFORM-audience, gated `ADMIN.TENANTS.CONFIGURE.GLOBAL`) writes three tables in a single request transaction:

1. `core.tenants` - the tenant row itself. `status='TRIAL'`, audit-actor pair populated from JWT.
2. `core.org_nodes` - the tenant-root row. `node_type='TENANT'`, `parent_id IS NULL`, `path` is an ltree label derived from `display_code` (if provided) or `name` (fallback). DDL CHECK `ck_org_nodes_root_parent_consistency` enforces the `node_type='TENANT' ↔ parent_id IS NULL` invariant.
3. `core.tenant_module_access` - one row per module in `modules_enabled` (ADMIN force-included by the request schema).

All three writes are atomic. A failure at any step rolls back the entire transaction; no partial tenants survive.

The tenant-root org_node row is the anchor for every tenant-scoped endpoint gated with `anchor_dep=get_tenant_anchor`. Its existence is a load-bearing invariant: omitting it causes every detail / write endpoint to 404 the tenant. Step 6.20.1 fixed the original Step 6.11.2 implementation which omitted this write; cleanup of orphan rows (POST-created tenants without org_nodes) was completed pre-fix.

### Slug derivation for tenant-root code / path

Pure-function helper in `repositories/tenants.py`. Input is `display_code` (if non-None) else `name`. Output is `(code, path)`:

- ASCII normalisation via `unicodedata.normalize('NFKD', ...)` + diacritic strip
- lowercase, non-alphanumerics collapsed to single `-`, leading/trailing `-` trimmed
- truncated to 64 chars (DDL CHECK `ck_org_nodes_code_format` max length)
- empty result → 422 `INVALID_TENANT_NAME_FOR_SLUG`
- `code` = result uppercased; `path` = result with `-` → `_` (ltree label requirement)

Examples:
- `Buc-ee's` → code `BUC-EE-S`, path `buc_ee_s` (mechanical slug; differs from seed's editorial `BUC-EES`)
- `Żabka Group` → code `ZABKA-GROUP`, path `zabka_group`
- `!!!` → 422

POST-created tenants use the mechanical slug. Seed-loaded tenants retain their curated codes (e.g. `BUC-EES`) because the seed loader inserts org_nodes directly with the Excel-specified code. A future PATCH-on-tenant-root surface would allow editorial override post-create.
