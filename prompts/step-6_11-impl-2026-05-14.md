## Step ID and intent

**Step 6.11** — Write endpoints for the `tenants` resource. Four
endpoints across two commits. **All four are platform-only.**

### Scope in

  `core.tenants` row (status forced to `TRIAL`) plus zero-or-more
  `core.tenant_module_access` rows in one transaction.

  Platform-only.
- `POST /api/v1/tenants/{tenant_id}/suspend` — TRIAL/ACTIVE →
  SUSPENDED.
- `POST /api/v1/tenants/{tenant_id}/activate` — TRIAL/SUSPENDED →
  ACTIVE.
- `require()` factory gains an `audience` kwarg.
- 4 new `ClientError` subclasses.
- `TenantsRepo` gains `.create()`, `.update()`, `.transition()`.
- Pydantic schemas: `TenantCreateRequest`, `TenantPatchRequest`.

  endpoint docs.
- `architecture_RBAC.md` two insertion points (Appendix A).
- CLAUDE.md, BUILD_PLAN.md, step doc updates.

## Locked decisions

1. **Two-commit WBS**: 6.11.1 (foundations) and 6.11.2
   (endpoints+docs).
2. **Endpoint surface**: 4 routes — POST `""`, PATCH
   `/{tenant_id}`, POST `/{tenant_id}/suspend`, POST
   `/{tenant_id}/activate`.
3. **Initial status on create**: `TRIAL`. Server-forced; DDL
   default `ONBOARDING` never hit in normal flow.
4. **ADMIN module force-included** in `tenant_module_access` rows.
5. **Tenant-module rows inserted in the same transaction** as the
   tenants row; `status='ENABLED'`,
   `enabled_by_user_id=auth.user_id`.
6. **Lifecycle**: create→TRIAL; TRIAL/ACTIVE→SUSPENDED on suspend;
   TRIAL/SUSPENDED→ACTIVE on activate. SUSPENDED→ACTIVE never
   re-enters TRIAL. Invalid transitions → 409
   `INVALID_STATE_TRANSITION`.
7. **PATCH allowed on SUSPENDED tenants**. `status` rejected by
   `extra="forbid"`.
8. **All four endpoints are platform-only**:
   - Layer 1: `audience="PLATFORM"` on `require()`.
   - Layer 2 permission tuples:
     - POST create + PATCH → `ADMIN.TENANTS.CONFIGURE.GLOBAL`
       (SUPER_ADMIN, PLATFORM_ADMIN).
     - POST suspend / activate → `ADMIN.TENANTS.OVERRIDE.GLOBAL`
       (SUPER_ADMIN only).
   - No `anchor_dep` (all scope=GLOBAL).
9. **Multi-audience PATCH deferred.** Lands post-6.16; see Scope
   out.
10. **`audience` parameter** added to `require()` as optional
    kwarg, default `None`. Existing call sites unchanged.
11. **Audit-actor pattern**: Pattern (a) per D-13. PLATFORM-only
    endpoint surface keeps the FK satisfied.
12. **Doc destination**: `architecture_RBAC.md` — Insertion 1
    (audience-kwarg paragraph) in gate-factory section; Insertion 2
    (3 worked examples) in `## Worked examples` section. Verbatim
    content in Appendix A.
13. **Doc bundling**: docs land in 6.11.2 with code.
14. **PATCH name-rename uniqueness check**. When `name` is in
    fields, `update()` runs the same SELECT-1 pre-check that
    `create()` uses, excluding self by id. Symmetric with create.
15. **Empty PATCH** → 422 `EMPTY_PATCH`. Handler-level check
    (after schema validation, before repo call).

```python
    number_of_stores_as_of_date: date | None = None
    modules_enabled: list[ModuleCode] = Field(default_factory=list)

    @field_validator("modules_enabled", mode="after")
    @classmethod
    def _force_include_admin(cls, v: list[ModuleCode]) -> list[ModuleCode]:
        """Dedupe preserving order; append ADMIN if absent."""

        result: list[ModuleCode] = []

                seen.add(m)
                result.append(m)
        if ModuleCode.ADMIN not in seen:
            result.append(ModuleCode.ADMIN)
```

```python
from enum import StrEnum
from typing import Any, Literal
from uuid import UUID
from admin_backend.errors import DuplicateTenantNameError


class TransitionResult(StrEnum):
    OK = "OK"
    NOT_FOUND = "NOT_FOUND"
    INVALID_STATE = "INVALID_STATE"


class TenantsRepo:
    # ... existing read methods unchanged ...

    async def create(
        self,
        *,
        name: str, region: str, tier: str, industry: str,
        country: str, primary_contact_name: str, contact_email: str,
        number_of_stores: int,
        display_code: str | None,
        monthly_revenue_usd: "Decimal | None",
        monthly_revenue_as_of_date: "date | None",
        number_of_stores_as_of_date: "date | None",
        modules_enabled: list[ModuleCode],
        actor_user_id: UUID,
    ) -> "TenantDetailRow":
        """Insert tenants row + tenant_module_access rows in one
        transaction. status=TRIAL. Caller dedupes and force-includes
        ADMIN. Raises DuplicateTenantNameError on name collision."""
        schema = get_settings().db_schema

        await self._raise_if_name_taken(name, exclude_tenant_id=None)

        new_tenant_row = await self.session.execute(
            text(f"""
                INSERT INTO {schema}.tenants
                    (name, region, tier, industry, country,
                     primary_contact_name, contact_email,
                     number_of_stores, display_code,
                     monthly_revenue_usd, monthly_revenue_as_of_date,
                     number_of_stores_as_of_date,
                     status, created_by_user_id, updated_by_user_id)
                VALUES
                    (:name, :region, :tier, :industry, :country,
                     :primary_contact_name, :contact_email,
                     :number_of_stores, :display_code,
                     :monthly_revenue_usd, :monthly_revenue_as_of_date,
                     :number_of_stores_as_of_date,
                     'TRIAL', :actor, :actor)
                RETURNING id
            """),
            {
                "name": name, "region": region, "tier": tier,
                "industry": industry, "country": country,
                "primary_contact_name": primary_contact_name,
                "contact_email": contact_email,
                "number_of_stores": number_of_stores,
                "display_code": display_code,
                "monthly_revenue_usd": monthly_revenue_usd,
                "monthly_revenue_as_of_date": monthly_revenue_as_of_date,
                "number_of_stores_as_of_date": number_of_stores_as_of_date,
                "actor": actor_user_id,
            },
        )
        new_tenant_id = new_tenant_row.scalar_one()

        # Per-module loop. Seed-scale module counts are 1-6 per
        # tenant; no PG array binds needed.
        for module in modules_enabled:
            await self.session.execute(
                text(f"""
                    INSERT INTO {schema}.tenant_module_access
                        (tenant_id, module, status,
                         enabled_at, enabled_by_user_id,
                         created_by_user_id, updated_by_user_id)
                    VALUES
                        (:tenant_id, :module, 'ENABLED',
                         now(), :actor, :actor, :actor)
                """),
                {"tenant_id": new_tenant_id, "module": module.value,
                 "actor": actor_user_id},
            )

        result_row = await self.get_by_id_with_aggregates(new_tenant_id)
        assert result_row is not None
        return result_row

    async def _raise_if_name_taken(
        self,
        name: str,
        *,
        exclude_tenant_id: UUID | None,
    ) -> None:
        """Raises DuplicateTenantNameError if any tenant other than
        `exclude_tenant_id` already has this name."""
        schema = get_settings().db_schema
        if exclude_tenant_id is None:
            row = await self.session.execute(
                text(f"SELECT 1 FROM {schema}.tenants WHERE name = :name LIMIT 1"),
                {"name": name},
            )
        else:
            row = await self.session.execute(
                text(f"SELECT 1 FROM {schema}.tenants WHERE name = :name AND id != :exclude_id LIMIT 1"),
                {"name": name, "exclude_id": exclude_tenant_id},
            )
        if row.first() is not None:
            raise DuplicateTenantNameError(name=name)

    async def update(
        self,
        tenant_id: UUID,
        *,
        fields: dict[str, Any],
        actor_user_id: UUID,
    ) -> "TenantDetailRow | None":
        """Partial update. Returns None if row missing or RLS-hidden
        (RLS-as-404 per D-17). Handler has verified fields non-empty.
        Runs name-uniqueness check on rename (locked decision 14)."""
        schema = get_settings().db_schema

        if "name" in fields:
            await self._raise_if_name_taken(
                fields["name"], exclude_tenant_id=tenant_id,
            )

        ALLOWED_UPDATE_KEYS: frozenset[str] = frozenset({
            "name", "display_code", "country", "tier", "industry",
            "primary_contact_name", "contact_email",
            "monthly_revenue_usd", "monthly_revenue_as_of_date",
            "number_of_stores", "number_of_stores_as_of_date",
        })
        invalid = set(fields.keys()) - ALLOWED_UPDATE_KEYS
        if invalid:
            raise ValueError(f"unexpected fields: {sorted(invalid)}")

        set_clauses = ", ".join(f"{k} = :{k}" for k in fields)
        set_clauses += ", updated_at = now(), updated_by_user_id = :actor"

        result = await self.session.execute(
            text(f"UPDATE {schema}.tenants SET {set_clauses} WHERE id = :tenant_id RETURNING id"),
            {**fields, "actor": actor_user_id, "tenant_id": tenant_id},
        )
        if result.first() is None:
            return None
        return await self.get_by_id_with_aggregates(tenant_id)

    async def transition(
        self,
        tenant_id: UUID,
        *,
        target_status: Literal["SUSPENDED", "ACTIVE"],
        actor_user_id: UUID,
    ) -> tuple["TenantDetailRow | None", TransitionResult]:
        """Atomic transition. SELECT FOR UPDATE, validate matrix,
        UPDATE, return post-state."""
        schema = get_settings().db_schema

        row = await self.session.execute(
            text(f"SELECT status FROM {schema}.tenants WHERE id = :tenant_id FOR UPDATE"),
            {"tenant_id": tenant_id},
        )
        current = row.first()
        if current is None:
            return None, TransitionResult.NOT_FOUND

        allowed_sources = {
            "SUSPENDED": {"TRIAL", "ACTIVE"},
            "ACTIVE": {"TRIAL", "SUSPENDED"},
        }[target_status]
        if current.status not in allowed_sources:
            return None, TransitionResult.INVALID_STATE

        if target_status == "SUSPENDED":
            await self.session.execute(
                text(f"""
                    UPDATE {schema}.tenants
                    SET status = 'SUSPENDED',
                        suspended_at = now(), suspended_by_user_id = :actor,
                        updated_at = now(), updated_by_user_id = :actor
                    WHERE id = :tenant_id
                """),
                {"actor": actor_user_id, "tenant_id": tenant_id},
            )
        else:
            await self.session.execute(
                text(f"""
                    UPDATE {schema}.tenants
                    SET status = 'ACTIVE',
                        suspended_at = NULL, suspended_by_user_id = NULL,
                        updated_at = now(), updated_by_user_id = :actor
                    WHERE id = :tenant_id
                """),
                {"actor": actor_user_id, "tenant_id": tenant_id},
            )

        result_row = await self.get_by_id_with_aggregates(tenant_id)
        return result_row, TransitionResult.OK
```

### Insertion 1 — `audience` parameter on `require()`

**Location**: inside the `require()` factory section, AFTER the
`anchor_dep` description and BEFORE `### Response envelope

````markdown
### Two-layer gate: optional `audience` parameter

The `require()` factory accepts an optional `audience` keyword:

```python
def require(
    module: ModuleCode,
    resource: PermissionResource,
    action: PermissionAction,
    scope: PermissionScope,
    *,
    anchor_dep: Callable | None = None,
    audience: Literal["PLATFORM", "TENANT"] | None = None,
) -> Callable[..., Awaitable[None]]:
```

When `audience` is set, the gate asserts `auth.user_type == audience`
BEFORE running `has_permission()`. On audience mismatch the gate
raises `PlatformAudienceRequiredError` (code
`PLATFORM_AUDIENCE_REQUIRED`, 403) or its tenant analogue, ahead of
the permission check.

This is defense-in-depth, not a replacement for the permission
check. A typical seeded catalogue already aligns grant audience
with scope. The audience parameter catches the catalogue-drift
case: if a future seed change ever leaks a `.GLOBAL`-scoped grant
to a TENANT-audience role, write endpoints with
`audience="PLATFORM"` continue to refuse at Layer 1.

`audience=None` (default) preserves all existing call sites
unchanged.

Endpoints exclusively administering platform-wide resources (e.g.,
the tenants write surface at Step 6.11) declare `audience="PLATFORM"`
on every route.

#### Worked example: writing the two-layer gate

```python
Depends(require(
    ModuleCode.ADMIN,
    PermissionResource.TENANTS,
    PermissionAction.CONFIGURE,
    PermissionScope.GLOBAL,
    audience="PLATFORM",
))
```

Order of checks at request time:

```
1. JWT verified by auth middleware           → 401 NOT_AUTHENTICATED
2. require() gate runs:
     a. audience check (Layer 1)             → 403 PLATFORM_AUDIENCE_REQUIRED
     b. has_permission(...) (Layer 2)        → 403 PERMISSION_DENIED
3. anchor_dep resolves (if set)              → 404 *NotFoundError
4. Handler runs.
```
````

````markdown
```python
@router.post("/{tenant_id}/suspend")
async def suspend_tenant(
    tenant_id: UUID,
    _: None = Depends(require(
        ModuleCode.ADMIN,
        PermissionResource.TENANTS,
        PermissionAction.OVERRIDE,
        PermissionScope.GLOBAL,
        audience="PLATFORM",
    )),
    auth: AuthContext = Depends(get_auth_context),
    session: AsyncSession = Depends(get_tenant_session_dep),
) -> TenantDetailResponse:
    """Transition TRIAL|ACTIVE → SUSPENDED.

    Gated on ADMIN.TENANTS.OVERRIDE.GLOBAL — held only by
    SUPER_ADMIN. PLATFORM_ADMIN can create/edit (CONFIGURE.GLOBAL)
    but cannot suspend/activate (OVERRIDE.GLOBAL). The catalogue
    encodes the privilege distinction.
    """
    repo = TenantsRepo(session)
    row, result = await repo.transition(
        tenant_id, target_status="SUSPENDED", actor_user_id=auth.user_id,
    )
    if result is TransitionResult.NOT_FOUND:

@router.post("/{tenant_id}/activate")
async def activate_tenant(
    tenant_id: UUID,
    _: None = Depends(require(
        ModuleCode.ADMIN,
        PermissionResource.TENANTS,
        PermissionAction.OVERRIDE,
        PermissionScope.GLOBAL,
        audience="PLATFORM",
    )),
    auth: AuthContext = Depends(get_auth_context),
    session: AsyncSession = Depends(get_tenant_session_dep),
) -> TenantDetailResponse:
    """Transition TRIAL|SUSPENDED → ACTIVE.

    Activate from SUSPENDED clears suspended_at and
    suspended_by_user_id atomically with the status flip. From
    TRIAL is forward-only progression.

    SUSPENDED → ACTIVE never lands back in TRIAL.
    """
    repo = TenantsRepo(session)
    row, result = await repo.transition(
        tenant_id, target_status="ACTIVE", actor_user_id=auth.user_id,
    )
    if result is TransitionResult.NOT_FOUND:
        raise TenantNotFoundError(tenant_id)
    if result is TransitionResult.INVALID_STATE:
        raise InvalidStateTransitionError(
            resource="tenant", tenant_id=tenant_id,
            target_status="ACTIVE",
        )
    return TenantDetailResponse.model_validate(row)
````
